# GPU 健康评分 v1 设计草案

本文档定义 Atlas 在第一阶段的 GPU 健康评分设计思路。目标不是直接依赖大模型完成预测，而是先建立“规则评分 + AI 解释增强”的可解释闭环，并为后续故障预测训练做数据准备。

## 1. 设计目标

第一阶段目标：

- 以 GPU 卡为最小对象输出健康评分
- 支持主机维度汇总评分
- 能识别高风险信号与潜在故障前兆
- 能解释评分原因和建议排查方向
- 为后续训练预测模型沉淀高质量样本

第一阶段不追求：

- 直接预测具体故障发生时间
- 纯大模型黑盒评分
- 一次性覆盖所有硬件类型

## 2. 数据来源

第一阶段重点依赖以下数据源：

- `DCGM Exporter`
  - GPU 温度
  - 功耗
  - 利用率
  - 显存利用率
  - 时钟
  - 降频原因
  - ECC 错误
  - XID 错误
  - PCIe 相关异常指标
- `Node Exporter`
  - CPU
  - 内存
  - 磁盘
  - 网络
  - 节点负载
  - 文件系统与 I/O 相关指标

建议保留的上下文字段：

- 机房 / 集群 / 机型
- 主机名
- GPU 型号
- GPU index / uuid
- 驱动版本
- CUDA 版本
- DCGM 版本
- 业务类型 / 训练任务类型

## 3. 评分对象

建议采用 3 层评分对象：

1. `GPU 卡评分`
   - 最核心对象
2. `主机评分`
   - 汇总主机内所有 GPU 以及节点层风险
3. `集群风险视图`
   - 用于横向对比和风险排序

## 5. 第一阶段评分方法

### 5.1 总体思路

第一阶段采用：

- 基础分 `100`
- 规则命中扣分
- 异常趋势附加扣分
- AI 基于命中结果生成解释和建议

即：

`最终评分 = 基础分 - 规则扣分 - 趋势扣分`

### 5.2 评分维度

建议至少覆盖以下维度：

- 温度健康
- 功耗健康
- 利用率匹配度
- 显存稳定性
- ECC / XID / 驱动异常
- 时钟与降频情况
- PCIe / 总线异常
- 同机横向偏差
- 历史基线漂移

### 5.3 规则类型

规则建议分成 4 类：

1. `绝对阈值规则`
   - 如温度过高、ECC 计数超限
2. `持续时间规则`
   - 如高温持续超过 15 分钟
3. `相对偏差规则`
   - 同机 8 张卡中某卡明显偏离
4. `趋势变化规则`
   - 过去 24 小时温度或错误率恶化

### 5.4 评分粒度

建议同时产出：

- 近 5 分钟即时评分
- 近 1 小时短周期评分
- 近 24 小时稳定性评分

最终展示可优先使用：

- 即时评分
- 24 小时趋势

## 6. 建议优先使用的风险信号

第一阶段建议优先抓以下高价值信号：

- 高温持续
- 同机温差异常
- 功耗高但利用率低
- 利用率大幅抖动
- 显存使用异常但吞吐不足
- XID 错误近期出现
- ECC 错误增速异常
- 长时间降频
- PCIe 异常重试或吞吐异常
- GPU 与节点资源状态不匹配

## 7. 为什么不能只看绝对值

GPU 健康判断不能只依赖阈值，必须结合：

- `同机对比`
  - 同一主机的多张卡之间找离群点
- `历史基线`
  - 相比自己过去 7 天 / 30 天是否劣化
- `多指标共振`
  - 温度、功耗、利用率、XID 联合异常更可信
- `业务上下文`
  - 不同训练负载下的正常区间不同

## 8. AI 在第一阶段的职责

AI 第一阶段不负责直接决定分数，主要负责：

- 解释为什么扣分
- 归纳可能根因
- 给出专业排查建议
- 生成日报 / 周报 / 风险摘要
- 输出是否需要更多日志和证据

建议 AI 输入包括：

- 命中的规则列表
- 核心指标摘要
- 同机对比结果
- 历史趋势摘要
- 最近错误事件
- 用户提供的相关日志片段

## 9. 为后续故障预测做准备

如果未来要做训练和预测，第一阶段就要开始沉淀训练资产。

### 9.1 必须沉淀的标签

- 正常样本
- 性能衰退样本
- 高温异常样本
- XID 异常样本
- ECC 异常样本
- 掉卡 / 失联样本
- 驱动 / 宿主机问题样本
- 散热 / 供电 / PCIe 问题样本

### 9.2 必须保留的时间窗口

针对每次故障事件，建议保留：

- 故障前 5 分钟
- 故障前 1 小时
- 故障前 6 小时
- 故障前 24 小时

### 9.3 推荐的中间特征

推荐先做统计特征，而不是直接把全量原始时序喂给模型：

- 均值
- 最大值
- 最小值
- 分位数
- 波动率
- 斜率
- 同机偏差
- 与历史均值偏差
- 超阈值持续时长
- 异常次数

## 10. 评分与预测的演进路线

### 阶段 A

- 规则评分
- AI 解释增强

### 阶段 B

- 异常检测
- 风险趋势建模

### 阶段 C

- 监督学习 / 半监督学习故障预测
- 将 AI 主要用于解释与报告，而不是直接替代预测模型

## 11. 数据表建议

建议后续增加以下模型：

- `gpu_metric_snapshots`
  - 原始或聚合指标快照
- `gpu_health_scores`
  - 每张卡评分结果
- `gpu_health_rule_hits`
  - 命中的规则明细
- `gpu_prediction_signals`
  - 预测输入特征与趋势信号
- `gpu_fault_labels`
  - 人工或规则确认的故障标签

## 12. 待确认项

- 是否确认 GPU 卡为第一阶段最小评分对象
- 是否确认第一阶段先用规则评分，不直接做纯模型预测
- 是否确认同机横向对比和历史基线作为核心评分逻辑
- 是否确认 AI 第一阶段只做解释、归因和建议
