# 故障 / 日志 Case 沉淀模板 v1

本文档用于统一沉淀故障处理、日志分析、GPU 异常排查等案例。目标是把历史经验从“零散笔记”转化为 Atlas 可检索、可复用、可训练、可用于 skills 的知识资产。

## 使用原则

- 一个 case 尽量只描述一类主要问题
- 原始现象与最终结论要分开写
- 结论必须尽量附带证据
- 能抽象为规则的内容，要单独写出
- 能抽象为 SOP 的内容，要单独写出
- 涉及敏感信息时先脱敏

## 精简版示例

### 1. 基本信息

- `case_id`: gpu-xid-0001
- `标题`: A800 单卡反复出现 XID 79 导致训练任务中断
- `日期`: 2026-04-14
- `作者 / 提供者`: chuan
- `状态`: 已确认

### 2. 适用范围

- `对象类型`: GPU
- `适用环境`: 生产
- `适用机型 / GPU 型号`: 8x A800
- `适用软件版本`: Driver 535.x / CUDA 12.x / DCGM exporter

### 3. 问题摘要

- `一句话描述`: 单张 GPU 间歇性掉卡并导致训练失败
- `故障等级`: 严重
- `影响范围`: 单机训练作业
- `是否可复现`: 否

### 4. 主要现象

- `用户侧现象`: 训练任务随机报 CUDA error
- `系统侧现象`: `nvidia-smi` 间歇性查询失败
- `监控异常`: 该卡功耗与利用率突然归零，温度曲线断崖下跌
- `是否伴随告警`: 是

### 5. 关键指标

- `关键监控指标`: 温度、功耗、SM 利用率、XID 错误
- `异常值 / 阈值`: 最近 1 小时内 XID 出现 3 次
- `持续时间`: 约 20 分钟内重复出现
- `同机 / 同批对比结果`: 同机其他 7 张卡正常

### 6. 关键日志

- `日志来源`: kernel / dmesg
- `关键错误关键词`: XID 79, GPU has fallen off the bus

### 7. 根因结论

- `最终根因`: 高概率为该卡 PCIe 链路或硬件稳定性异常
- `根因类型`: PCIe
- `证据`: XID 79、掉卡、同机其他卡正常、训练任务仅在该卡失败
- `置信度`: 高

### 8. 处理动作

- `临时止血动作`: 下线该卡并迁移任务
- `永久修复动作`: 安排现场复查插槽、供电与硬件状态，必要时更换
- `处理结果`: 迁移后任务恢复正常

### 9. 可沉淀资产

- `可抽象规则`: 单卡出现 XID 79 且功耗/利用率归零时标记高风险
- `可抽象 SOP`: 先隔离卡，再检查 PCIe、供电、硬件连接
- `可抽象日志模式`: `GPU has fallen off the bus`
- `可作为 AI few-shot 示例`: 是
- `可作为训练样本`: 是
